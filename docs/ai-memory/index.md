# What2Pick AI Memory Index

Read this file and `current-state.md` at the start of every What2Pick work session.

## Files

- `current-state.md`: current confirmed state, active facts, known gaps, and latest session notes.
- `project-overview.md`: confirmed stack, repository structure, commands, and product requirements.
- `decisions.md`: important product and technical decisions.
- `backlog.md`: planned, proposed, and unknown work items.

## Memory Rules

- Code and tests are the source of truth.
- Consult other memory files only when relevant.
- Clearly label functionality as implemented, planned, proposed, or unknown.
- Keep notes brief and factual.
- Update memory after important decisions, meaningful progress, blockers, or before ending a session.
- Do not store passwords, tokens, keys, `.env` files, personal data, full conversations, or detailed command logs.
