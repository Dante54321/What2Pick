# What2Pick Agent Instructions

## Project Memory

- At the start of a What2Pick work session, read `docs/ai-memory/index.md` and `docs/ai-memory/current-state.md` first.
- Read the other files in `docs/ai-memory/` only when they are relevant to the task.
- Treat the current code and tests as the source of truth.
- Clearly distinguish implemented, planned, proposed, and unknown functionality.
- Update project memory after an important decision, meaningful progress, a blocker, or before ending a work session.
- Keep memory notes brief and clear. Do not copy full conversations or create a detailed command log.
- Never store passwords, tokens, keys, `.env` files, or personal data in memory.
- Do not create `.codex/config.toml` unless explicitly requested.
- Do not copy internal Codex skills into this repository.
